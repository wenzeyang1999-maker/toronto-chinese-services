> 背景：老板提出"取消地图模式"。经分析，地图在平台有 4 种用途，价值差别很大，建议**按场景精简而非全砍**（详见 `docs/地图去留-利弊分析.pdf`）。以下按推荐范围拆解。

- [x] **D1. 列表优先，地图可选（不再默认地图）** — ✅ 已完成（2026-07-13）。按最终定调：**保留地图切换按钮**，只把首页「找服务」的默认视图从地图改为列表；用户手动切到地图仍会被记住（localStorage）。全站列表页现已统一为"列表默认、地图可选"（Category/Search/RealEstate 本就是列表默认）。tsc 0 错误。
  - 改动文件：`src/pages/Home/Home.tsx`（默认 `'map'` → `'list'`）
- [ ] **D2. 保留三处高价值地图（确认保留，无需开发）** — ① 房产找房地图 ② 服务详情「位置 + 一键导航」③ 商家「发现客户」看附近需求。此三处砍掉会伤核心工作流，默认不动。
- [ ] **D3.（可选 · 降本）服务详情交互地图 → 静态图 + 导航链接** — 若主要动机是压 Google Maps API 成本，把服务详情的交互式地图换成静态缩略图 + 一键导航 deeplink，费用大降、体验基本不掉。｜粗估 0.5–1 天
