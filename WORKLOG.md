# 开发工作日志 — Toronto Chinese Services

## 2026-03-26

### 服务数据接入 Supabase 数据库
- **发布服务写入数据库**：PostService 表单提交后，数据正式保存到 Supabase `services` 表，不再只存在于内存
- **首页/搜索读取真实数据**：App 启动时调用 `fetchServices()`，首页和搜索页展示的是数据库里的真实服务
- **登录校验**：未登录用户点击发布服务，自动跳转到登录页
- **用户资料同步**：发布服务时自动更新用户的姓名、电话、微信号到 `users` 表
- **图片上传框架**：代码层面已支持上传最多 3 张图片到 Supabase Storage（需配置 `service-images` bucket，待后续完善）

### 数据库权限修复
- 修复 `users` 表 RLS 策略：改为公开可读，解决首页加载服务商信息报错问题
- 补充 `services` 表 INSERT 权限，允许登录用户发布服务
- `services` 表新增 `images TEXT[]` 列存储图片链接
- 补充 `other`（其他服务）分类到数据库

### Schema 完善（一次部署即可用）
- 所有建表、策略、触发器语句加上防重复 guard，新建 Supabase 项目只需跑一次 `schema.sql`

## 历史工作（2026-03-25 及之前）

### 首页 & 响应式布局
